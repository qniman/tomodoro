# Трешшутинг — типовые проблемы и решения

## 1. Ошибка миграций `SQL` или `permission denied`
- Убедитесь, что DB доступна и параметры в `.env` корректны.
- Проверьте права пользователя БД и кодировку.

## 2. Пустая страница после деплоя / 500
- Проверьте логи: `storage/logs/laravel.log`.
- Убедитесь, что `APP_KEY` задан и `php artisan config:cache` выполнен.

## 3. Очереди не работают
- Проверьте, запущен ли worker (`php artisan queue:work`) и соединение с очередью (Redis).
- Посмотрите логи очередей.

## 4. npm / vite сборка упала
- Установите зависимости заново: `rm -rf node_modules && npm ci`.
- Проверьте Node.js версию.

## 5. Проблемы с правами к папкам
- Выполните:
```
sudo chown -R www-data:www-data storage bootstrap/cache
sudo chmod -R 775 storage bootstrap/cache
```

## 6. Почта не уходит
- Убедитесь в настройках `MAIL_*` и проверьте логи почты/SMTP.

Если нужна помощь с конкретной ошибкой — приложите лог и описывайте шаги для воспроизведения.
